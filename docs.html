Objetivos

O objetivo deste trabalho é desenvolver um compilador-tradutor de uma linguagem hipotética para codigo em linguagem C, implementado inteiramente em Client-Side Javascript. A tecnologia Javascript permite que nosso compilador seja executado em qualquer plataforma, desde que para tal plataforma exista um navegador web que ofereça suporte a esta linguagem.

A linguagem Javascript é particularmente interessante pois é uma linguagem dinâmica, e, como tal, permite uma maior liberdade em implementação ao custo de uma certa perda em eficiência. Desvantagem esta que, com o advento das últimas tecnologias incorporadas nos navegadores mais recentes, tem se tornado cada vez menos relevante, tendo em vista que as novas máquinas virtuais JavaScript aceleram e otimizam código transformando-o em código de máquina nativo antes de sua execução.

Toda implementação foi feita utilizando-se Orientação a Objetos, facilitando o desenvolvimento e ao mesmo tempo fornecendo um exemplo de como aplicar esta técnica na linguagem JavaScript.

Linguagem

A linguagem ALG é uma linguagem hipotética com estrutura similar ao Pascal, procedural, sem muitos recursos, porém com instruções suficientes para criação de algoritmos simples. A especificação da linguagem através de uma gramática livre de contexto será disponibilizada em breve.

Análise Léxica

O principal componente responsável por conduzir a analise lexicográfica é a classe AnaliseLexica.js. A partir de uma entrada em texto qualquer, esta classe é responsavel por identificar os tokens presentes nesta entrada na ordem em que aparecem, utilizando seu método principal getToken().

Cada Token consiste basicamente num par contendo um lexema (ou semantema) e a categoria do lexema identificado, como, por exemplo, (42,numero_inteiro) ou (myVar,identificador). Posteriormente, informações adicionais como a linha em que o lexema foi encontrado terão de ser adicionadas para as próximas análises.

A tabela de palavras reservadas é implementada na classe Keywords. Como internamente, todo objeto em JavaScript é implementado como uma hashtable, pudemos contar com este dinamismo da linguagem como vantagem para implementar a tabela de palavras reservadas como um simples objeto e ainda assim obter a eficiência de busca desejada.

Mais informações sobre a análise léxica em compiladores podem ser encontradas na Wikiversidade, a universidade livre. Clique no primeiro link para ir direto ao tópico.

Análise Sintática

A classe responsável por conduzir a análise sintática é a AnalisadorSintatico.js. A partir de uma entrada qualquer, esta classe instancia um analisador léxico e começa a utilizá-lo para identificar a sintaxe do texto, a partir das regras sintáticas da linguagem ALG, implementadas a partir de gráfos sintáticos.

Uma das maiores dificuldades encontrada nesta fase é o tratamento de erros. A estratégia implementada é o modo pânico, que consiste em não tentar corrigir o erro e tentar seguir a análise adiante apenas relatando o erro encontrado.
